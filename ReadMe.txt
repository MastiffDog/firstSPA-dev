***Приложение СуперБлог***
Реализованный функционал:
1. На начальной стадии посетителю доступны вкладки
   - о создателе;
   - пользователи;
   - статьи;
   - главная;
2. Вкладки "Пользователи" и "Статьи" позволяют просматривать списки зарегистрированных пользователей,
   смотреть комментарии (содержания статей нет на https://jsonplaceholder.typicode.com/, поэтому сделан начальный объект
   на 10 статей с применением "рыботекста"). Ставить "лайки", "дизлайки" и комментировать эти статьи нельзя без входа на сайт.
3. Регистрация нового пользователя: в форме регистрации реализована проверка на заполнение всех полей, без этого
   пользователь не будет зарегистрирован. Также есть проверка на повторную регистрацию пользователей с одинаковыми именами.
   Можно добавить проверку мейла по регулярному выражению, но т.к. без входа пользователя приложение полноценно не посмотреть,
   я решил не добавлять это дело для удобства просмотра функций. Проще всего в форме регистрации ввести 4 одинаковых цифры и смотреть
   дальше логику работы.
4. Режим "пользователь вошел". Зайти можно под именем любого пользователя из тех 10, что загружаются
   с сайта https://jsonplaceholder.typicode.com/ (можно посмотреть в массиве usernames, файл first_state.js в константах),
   либо можно сначала зарегистрировать пользователя (регистрация формальная).Для входа в полях логина и пароля пишется имя пользователя
   (для простоты). Когда вошел пользователь, в меню навигации появляется возможность добавить свою статью, появляется галерея рисунков
   (подгружаются), можно при прочтении статьи писать комментарии (неограниченно), лайкнуть-дизлайкнуть можно только 1 раз, и либо лайк,
   либо дизлайк. После появления лайка-дизлайка соответствующие кнопки исчезают.
   Чтобы увеличить число лайков статьи, необходимо зайти под именем другого пользователя (как в реальной жизни).
5. Режим "вошел администратор". Логин-пароль: admin. Все то же самое, что и для пользователя, только зайти можно с
   самого начала.
6. Режим добавления статьи. Пишем статью, она после добавления будет первой в статьях. Пользователю, который добавил статью,
   разрешено только смотреть комменты и писать свои, лайкнуть-дизлайкнуть свой труд он не имеет права.

***ДОПУЩЕНИЯ:

-  т.к. у нас есть 2 вида пользователей - подгружаемые, и те, которых мы создаем сами во фронте, а также админ,
   то сделан массив с их именами, чтобы не писать отдельную функцию, которая из всего этого хозяйства будет делать один массив.
   Понятно, что в реальной жизни будет написан один асинхронный запрос на сервер, и там все эти данные будут в одном месте.

- в редьюсере, который отвечает за изменение стейтов при успешной загрузке списка пользователей сделана проверка на одноразовую
  загрузку пользователей, и если мы их один раз успешно получили, дальше мы не используем payload из action. Причина:
  нам надо собрать единый массив users, который должен содержать только уникальных пользователей, без повтора. Если бы был
  сервер, то мы бы сильно упростили себе жизнь, т.к. там было бы единое место хранения юзеров.

- комментарии пользователей. Поскольку на сайте https://jsonplaceholder.typicode.com/comments отсутствует разбивка по
  количеству комментариев, и асинхронный запрос подгружает сразу 500 комментов, то не вижу смысла делать запросы из серии "показать
  следующие 10 и т.п.". Потому что физически это будет один и тот же запрос. В реальности, насколько я вижу из типа json
  файла, там есть ID поста и ID коммента. Соответственно, на реальном сервере это будут уже несколько разных запросов.
  В статьях для читабельности я ограничил вывод 5 комментариями, чтобы можно было легко читать комментарии добавляемые
  зарегистрированными пользователями, либо админом.

- галерея. Была мысль сделать галереи пользователей, добавление изображений и т.п., но функционал будет повторять функционал статей,
и без бэка это просто трата времени. Поэтому подгружаются урлы с сайта, массив обрезан до 10 фото, и их можно посмотреть.
